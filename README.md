# 이음(理音) 모바일웹

후원자·협의체 위원·운영자가 비식별 케이스와 지역 후원을 안전하게 연결하는 1차 운영 모바일웹입니다.

## 목표

- 1차: 모바일웹/PWA 운영 완성
- 2차: 동일 API와 상태 모델을 사용하는 Flutter 앱
- 대상자는 앱 계정을 만들지 않음
- 후원자에게 대상자의 실명·전화번호·상세 주소를 공개하지 않음

## 구조

```text
apps/web   Next.js 모바일웹
apps/api   FastAPI 업무 API
packages   공통 계약과 디자인 토큰
docs       상태·권한·개인정보·운영 문서
infra      배포와 백업 구성
```

## 첫 수직 기능

```text
후원 등록 → 운영 검토 → 보완/반려/승인 → 연결 목록 노출
```

### 구현 완료

- JWT 로그인과 조직·역할 권한
- 후원 등록 `PENDING_REVIEW`
- 운영 승인 대기 목록
- 승인·보완 요청·반려
- 같은 후원 ID 수정·재제출
- 승인 전 연결 목록 비노출
- 승인 후 `AVAILABLE` 연결 목록 노출
- 처리자·상태·사유·일시 감사 로그
- 기한이 지난 후원 등록 차단
- 모바일 6자리 사용자 번호 간편 로그인과 서버 확인 역할별 라우팅
- 후원자 내 후원·등록·보완 재제출 화면
- 운영자 승인·보완·반려 검토함
- 협의체 위원 연결 가능 후원·카테고리 필터
- 계정/IP DB 공유 로그인 제한과 감사 가능한 상태 전이

정적 웹 미리보기: https://kevinsaem.github.io/ieum-mobile-web/

현재 GitHub Pages는 UI 미리보기이며, 운영 API는 별도 HTTPS 서버에 배포해야 합니다. API 주소가 없는 Pages 빌드는 로그인 정보를 네트워크로 전송하지 않습니다.
