# 1차 인수인계 인수 조건

## 시나리오 A: 후원 승인

```gherkin
Given 정담식당 후원자가 로그인했다
When 반찬 10세트와 사진 3장을 검토 요청한다
Then 상태는 PENDING_REVIEW 이다
And 연결 목록에는 노출되지 않는다

Given 운영자가 해당 후원에 보완을 요청했다
When 후원자가 같은 ID를 수정해 재제출한다
And 운영자가 승인한다
Then 상태는 AVAILABLE 이다
And 협의체 위원의 연결 목록에 노출된다
And 모든 상태 변경이 감사 로그에 남는다
```

## 시나리오 B: 케이스 승인과 추천

```gherkin
Given 위원이 동의 근거를 확인하고 비식별 케이스를 제출했다
Then 상태는 PENDING_REVIEW 이다
And 추천 API는 접근을 거절한다

Given 운영자가 케이스를 승인했다
Then 상태는 SEARCHING 이다
And 승인된 AVAILABLE 후원만 추천한다
```

## 시나리오 C: 후원자 확약과 공동 승인

```gherkin
Given 위원이 매칭을 신청했다
Then 후원자는 확정 수량과 픽업 시간을 확인해야 한다
And 제공 확약 전 최종 재고 예약은 불가능하다

Given 운영 적정성 검토가 통과했다
When 서로 다른 승인자 2인이 승인한다
Then 재고는 한 번만 예약된다
And 동일 승인자 중복 승인은 거절된다
```

## 시나리오 D: 부분 전달

```gherkin
Given 10세트가 예약되었다
When 7세트만 전달되었다
Then 7세트는 완료로 기록된다
And 잔여 3세트 필요는 재매칭 가능하다
And 후원 잔여 수량과 재고 원장이 일치한다
```

## 시나리오 E: 개인정보 경계

```gherkin
Given 후원자가 케이스와 매칭 상세를 조회한다
Then 실명·전화번호·상세 주소·상담 원문이 응답에 없어야 한다
And 사진은 권한이 있는 짧은 만료시간 URL로만 조회한다
```
